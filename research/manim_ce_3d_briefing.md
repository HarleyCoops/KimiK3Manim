## 2. Renderers: Cairo vs OpenGL
- Cairo is the default, CPU-bound, most-tested renderer. All docs/examples assume it; it is what "3D in CE" is designed around. OpenGL (--renderer=opengl) uses moderngl shaders on GPU and is faster, with a live preview window and self.interactive_embed() for an IPython session inside the running scene.
- CE's own FAQ concedes OpenGL classes (OpenGLMobject, OpenGLVMobject, OpenGLSurface) are NOT in the online docs — docstrings only. interactive_embed can hit sqlite3.ProgrammingError with recent IPython (workaround: IPython==8.0.1).
- Feature gaps historically: Surface animations broken in OpenGL (issue #2756, fixed 0.19.1 #4286); Prism/Cube/Code fixes landed only in 0.19.1. Verdict: OpenGL in CE 0.19 is usable and improving but still second-class — not "production-ready" as a default for final 3D renders. It is excellent for interactive iteration (camera scouting) and roughly 3b1b-style workflows, but final pixel output should default to Cairo. To write video from OpenGL you need --write_to_movie; GUI flags: --enable_gui, --enable_wireframe, --force_window, --use_projection_fill_shaders/--use_projection_stroke_shaders.
Sources: https://docs.manim.community/en/stable/faq/opengl.html ; https://docs.manim.community/en/stable/guides/configuration.html ; https://github.com/ManimCommunity/manim/issues/2756 ; https://www.aquabeam.me/manim/opengl_guide/

## 3. 3D scene machinery
Module: manim.scene.three_d_scene.ThreeDScene; camera: manim.camera.three_d_camera.ThreeDCamera (manim/camera/three_d_camera.py).
- ThreeDCamera ctor: focal_distance=20.0, shading_factor=0.2, default_distance=5.0, light_source_start_point=9*DOWN+7*LEFT+10*OUT, should_apply_shading=True, exponential_projection=False, phi=0, theta=-90*DEGREES, gamma=0, zoom=1.
- phi/theta/gamma/zoom/focal_distance are ValueTrackers (phi_tracker, theta_tracker, ...) — so camera motion is animatable via self.play(self.camera.theta_tracker.animate.set_value(...)) or the higher-level move_camera(...).
- ThreeDScene API: set_camera_orientation(phi,theta,gamma,zoom,focal_distance,frame_center) — instant; move_camera(same kwargs + added_anims=[], **kwargs e.g. run_time) — animated; begin_ambient_camera_rotation(rate=0.02, about="theta"|"phi"|"gamma") / stop_ambient_camera_rotation(); begin_3dillusion_camera_rotation(rate=1, origin_phi, origin_theta) / stop_3dillusion_camera_rotation(); set_to_default_angled_camera_orientation(**kwargs); add_fixed_in_frame_mobjects (HUD overlays), add_fixed_orientation_mobjects(*mobs, use_static_center_func, center_func) (billboard labels that keep facing the camera).
- Projection: points are rotated by rotation_matrix built from phi/theta/gamma, then perspective-scaled by focal_distance/(focal_distance - z); exponential_projection=True smooths the z>=focal_distance singularity. Default camera (phi=0, theta=-90deg) looks straight down -Z at the xy-plane, i.e. a flat 2D view — the #1 "my 3D looks 2D" gotcha; always call set_camera_orientation(phi=70*DEGREES, theta=-45*DEGREES)-style.
- MovingCameraScene (2D, manim.scene.moving_camera_scene) uses self.camera.frame; CE ships no combined moving+3D camera scene — in 3D you animate via move_camera or camera ValueTrackers instead.
Sources: https://docs.manim.community/en/stable/reference/manim.scene.three_d_scene.ThreeDScene.html ; https://docs.manim.community/en/stable/reference/manim.camera.three_d_camera.ThreeDCamera.html ; https://github.com/ManimCommunity/manim/blob/v0.19.2/manim/camera/three_d_camera.py

## 4. 3D mobject catalog + painter's algorithm
Module manim.mobject.three_d.three_dimensions: Surface, Sphere, Torus, Cone, Cylinder, Cube, Prism, Arrow3D, Line3D, Dot3D, ThreeDVMobject. Module manim.mobject.three_d.polyhedra: Polyhedron, Tetrahedron, Octahedron, Dodecahedron, Icosahedron, ConvexHull3D (new in 0.19.0). Axes: ThreeDAxes in manim.mobject.graphing.coordinate_systems.
- ParametricSurface does NOT exist in 0.19 — renamed to Surface back in v0.10.0 (old code using ParametricSurface fails).
- Surface(func, u_range=(0,1), v_range=(0,1), resolution=32, fill_color=BLUE, fill_opacity=1.0, checkerboard_colors=[...], stroke_color, stroke_width=0.5, should_make_jagged=False, surface_piece_config={}, pre_function_handle_to_anchor_scale_factor=1e-5). It is a VGroup of per-face VMobjects; resolution is int or (u_res, v_res). checkerboard_colors OVERRIDES fill_color — pass checkerboard_colors=False or set fill after creation. Cost scales with faces = u_res*v_res submobjects.
- Surface.set_fill_by_checkerboard(*colors, opacity); Surface.set_fill_by_value(axes, colorscale=[(RED,-0.5),(YELLOW,0),(GREEN,0.5)], axis=2) — value-mapped gradient with pivots (underused, great for "epic" looks).
- Sphere/Torus/Cone/Cylinder subclass Surface; Cube/Prism are VGroups (Cube got bevel line joints in 0.19.1); Line3D is a cylinder between endpoints; Arrow3D = cylindrical shaft + conical tip; Dot3D = shaded ball (has shade_in_3d semantics).
- Painter's algorithm (Cairo): ThreeDCamera.get_mobjects_to_display sorts whole mobjects by depth key = dot(mob.get_z_index_reference_point(), rotation_matrix.T)[2]; mobjects WITHOUT shade_in_3d get key=+inf (always drawn last = on top). Shading (modified_rgbas) darkens start/end corner colors via get_shaded_rgb against camera.light_source (a Point mobject you can .move_to()). Artifacts: sorting is per-mobject, not per-face/triangle — intersecting surfaces, surfaces threading through each other, or a large face vs small nearby object will pop or draw in the wrong order; strokes on every face of a Surface produce visible grid seams at high zoom. Fixes: avoid interpenetrating geometry; split surfaces; nudge with .set_z_index()/z sorting; set stroke_width=0; or use the OpenGL renderer (true z-buffer).

## 5. "Epic look" techniques in the Cairo renderer
- Gradients: VMobject.set_color_by_gradient(*colors); per-value surface gradients via Surface.set_fill_by_value; 3D shading via shade_in_3d=True (ThreeDVMobject default) + camera light. Sheen (2D specular sweep): VMobject.set_sheen(factor, direction) / sheen_factor — cheap "premium" highlight.
- Glow/bloom: NO built-in Glow/bloom mobject in CE (unlike manimgl's Glow). Emulate: layered strokes (same path, increasing stroke_width with decreasing stroke_opacity), duplicate scaled copies at low opacity, or radial-gradient bitmap via VMobject(background_image=...). True bloom = render passes and composite externally.
- Shadows/reflections/DOF/fog: none built-in. Fake contact shadows with flattened dark semi-transparent copies under objects; fake depth fog with an updater mapping camera-distance -> opacity/desaturation; fake reflections with mirrored low-opacity copies on a floor plane.
- Camera fly-throughs: move_camera(frame_center=..., phi=..., theta=..., focal_distance=..., run_time=...) plus ambient rotation; small focal_distance (~2-5) gives wide-angle drama; exponential_projection=True avoids blowups near the focal plane.
- Parametric motion: ValueTracker + add_updater(lambda mob, dt: ...); turn_animation_into_updater(anim, cycle, delay) (delay new in 0.19); always_redraw for live-rebuilt surfaces (expensive; keep resolution low).
- Morphs between surfaces: build both as Surface with identical resolution and Transform(a, b) (point counts must match; use same resolution tuple). No TransformBetweenSurfaces class — that's just Transform on Surfaces.
- Compositing passes: render overlays with -t/--transparent (.mov/.webm alpha) and composite in ffmpeg/Resolve; CE has no multi-pass pipeline.

## 6. Performance engineering
- Quality presets (-q): l=854x480@15fps, m=1280x720@30, h=1920x1080@60, p=2560x1440@60, k=3840x2160@60; custom: -r W,H and --fps. Iterate at -ql, ship at -qh/-qk. -s saves last frame; -n m,n renders an animation slice; --save_sections splits per-section files.
- Cost drivers: Surface resolution (submobject count), n_points_per_cubic_curve, per-face strokes, always_redraw surfaces, 4K60 pixel count. resolution=32 surface = ~1024 faces; 64 = ~4096 — Cairo render time scales roughly linearly.
- Caching: manim hashes each play() call and reuses partial movie files when code is unchanged (partial_movie_dir); flags --disable_caching, --flush_cache, max_files_cached. v0.19.0 added a separate writer thread for frame output (PR #3888).
- Parallelism: no built-in multi-process rendering of one scene; parallelize by rendering different scenes or -n slices in separate manim processes, then concatenate (PyAV bundled since 0.19, no system ffmpeg needed).
- When to drop to manimgl (3b1b): true GPU z-buffered 3D, custom GLSL shaders, smooth continuous surfaces, live interactive camera. Friction: import manimlib, different API/kwargs/defaults, undocumented breaking changes, no CE plugin compat, hand-port scenes.

## 7. Extension points for an AI scene generator
- Scene surface: construct(), self.play(*anims, run_time=), self.wait(), self.add/remove, self.next_section(section_type=) (renamed from type= in 0.19.0), self.time (0.19+), self.camera (ThreeDCamera), .animate method-builder (mob.animate.shift(UP)), Animation.set_default() (0.19).
- Mobject anatomy: .points ndarray (n_points_per_cubic_curve=4 on VMobject), submobjects/family, add_updater/remove_updater/suspend_updating, become(), set_z_index, get_z_index_reference_point.
- Most useful 3D animations: Create/Write, FadeIn/FadeOut, Transform/ReplacementTransform, Rotate(mob, angle, axis=OUT|RIGHT|UP, about_point=), Rotating(axis=..., radians=..., run_time=), ApplyMatrix, MoveAlongPath, GrowFromCenter, Add (0.19), camera anims via move_camera/ValueTrackers.
- LaTeX in 3D: Tex/MathTex are flat; billboard with add_fixed_orientation_mobjects(tex) or pin as HUD with add_fixed_in_frame_mobjects. MathTex rewritten in v0.20 (more robust splitting); LaTeX color commands preserved since 0.19.0.
- Common LLM failure modes + fixes: (1) ParametricSurface -> use Surface; (2) 3D looks flat -> set_camera_orientation(phi=..., theta=...); (3) fill_color ignored on Surface -> checkerboard_colors overrides it; (4) interpenetrating surfaces flicker/misorder -> painter's algorithm, restructure geometry or set_z_index; (5) Tex drifting with camera -> fixed-in-frame/orientation; (6) resolution too high -> slow renders; 16-32 while iterating; (7) config.renderer="opengl" missing features -> stay Cairo for final; (8) mixing manim/manimgl imports -> manim=CE, manimlib=3b1b; (9) Scene.next_section(type=) -> section_type=; (10) camera angles must be radians (use DEGREES constant).

## 9. Install & coexistence (verified)
- PyPI package: manim (CE). Install: pip install manim — PyAV bundles ffmpeg since 0.19, no system ffmpeg needed; docs now recommend uv (uv tool install manim or uv add manim). Conda/Docker optional. LaTeX still required for Tex/MathTex.
- manimgl coexistence: PyPI manimgl (3b1b) installs module manimlib + its OWN manim executable that can shadow CE's. Libraries coexist (different import names); CLI shims conflict. Fixes: use manimce executable or python -m manim; check manim --version prints "Manim Community v...". PyPI manimlib = ancient ManimCairo, only for re-rendering pre-2021 3b1b projects.
